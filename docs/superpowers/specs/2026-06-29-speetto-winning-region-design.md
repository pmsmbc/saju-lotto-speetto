# 스피또 당첨 지역 조회 — 설계 문서

작성일: 2026-06-29

## 배경 / 목표

사주 + 한국 로또/스피또 정보를 제공하는 사이트의 **1차 기능**으로,
동행복권의 스피또(즉석복권) 당첨 판매점 데이터를 받아와
**회차별로 → 지역별 집계**를 보여주는 메뉴를 만든다.

- 프론트엔드: React (Vite)
- 호스팅: GitHub Pages (정적), 완전 무료, 서버 없음
- 데이터 수집: GitHub Actions 예약 작업(스크래퍼) → JSON 저장

이후 단계(이 문서 범위 밖): 로또 번호 추천, 오늘의 띠별 추천 번호, 생년월일시(사주) 기반 번호 추천.
앱 셸(메뉴 바)에 자리만 만들어 두고 "준비중"으로 표시한다.

## 전체 구조

```
[GitHub Actions 봇] --하루 1~2회--> 동행복권 스크래핑 --> public/data/speetto.json (커밋)
                                                              │
[사용자] --> GitHub Pages (정적 React 사이트) --읽기--> speetto.json --> 화면 표시
```

- 서버 없음. 데이터 수집은 GitHub Actions(예약 작업), 화면은 정적 React.
- 스피또 데이터는 자주 안 바뀌므로 하루 1~2회 갱신으로 충분.

## 데이터 수집 (스크래퍼)

- 위치: `scripts/scrape-speetto.js` (Node.js)
- 대상: 현재 판매 중인 스피또 게임(스피또2000 / 1000 / 500)의 당첨 판매점 데이터.
- 동작:
  1. 동행복권 페이지/내부 요청 호출
  2. 당첨 판매점 항목 파싱
  3. 주소에서 시·도(지역) 추출
  4. JSON 저장
- 구현 시점에 동행복권의 실제 응답 형태(POST/JSON/HTML)를 직접 확인해
  가장 안정적인 방법을 사용한다.
- 실패/차단/데이터 없음 시: 마지막으로 성공한 JSON을 유지하고, 화면에서 안내 처리.

### 결과 파일 형식: `public/data/speetto.json`

```json
{
  "updatedAt": "2026-06-29T03:00:00Z",
  "stores": [
    {
      "game": "스피또1000",
      "round": "78회",
      "rank": 1,
      "store": "행운편의점",
      "address": "서울 강남구 ...",
      "region": "서울"
    }
  ]
}
```

- 각 항목은 `game`, `round`(회차), `rank`(등수), `store`(판매점명), `address`(주소), `region`(시·도)을 가진다.
- `region`은 화면 집계의 기준이 되며, 주소 앞부분에서 표준 시·도 명으로 정규화한다.

## 자동 실행 (GitHub Actions)

- 파일: `.github/workflows/scrape.yml`
- 트리거:
  - 매일 정해진 시간(예: 오전 9시 KST) 예약 실행
  - `workflow_dispatch` 수동 실행 버튼
- 동작: 스크래퍼 실행 → JSON 변경 시 자동 커밋 → GitHub Pages 재배포.

## 화면 (React)

- 도구: Vite + React
- 스타일: 가벼운 기본 CSS (외부 UI 프레임워크 없음)
- 앱 셸: 상단 메뉴 바. 현재는 "스피또 당첨 지역"만 활성화.
  나머지(로또 추천 / 띠별 번호 / 사주 번호)는 "준비중" 표시.

### 스피또 당첨 지역 페이지 구성 (회차별 → 지역별 집계)

1. 상단: "마지막 업데이트: YYYY-MM-DD" 표시
2. **회차 선택**: 게임/회차 드롭다운 (예: "스피또1000 78회", 또는 "전체")
3. **선택한 회차의 지역별 집계**: "서울 5 · 경기 3 · 부산 1 ..." (당첨 많은 순)
4. **지역 필터**: 지역 클릭 시 그 지역만 보기
5. **당첨 판매점 목록**: 등수 / 판매점명 / 주소 (선택한 회차 + 지역 기준)

## 컴포넌트 분리

- `App` — 메뉴 바 + 라우팅
- `SpeettoPage` — 데이터 로딩 + 페이지 조립
- `RoundSelector` — 게임/회차 선택
- `RegionStats` — 선택된 회차 기준 지역별 집계 표시
- `StoreList` — 선택된 회차 + 지역 필터 적용 목록
- `useSpeettoData` — JSON 불러오는 훅 (로딩/에러 처리)

## 에러 처리

- JSON 로딩 실패 시: "데이터를 불러올 수 없습니다" 안내
- 데이터 비어있을 때: "현재 표시할 당첨 정보가 없습니다"
- 스크래퍼 실패해도 기존 JSON은 유지 → 화면은 계속 정상 동작

## 범위 밖 (이번 작업에 포함하지 않음)

- 로또 번호 추천, 띠별 추천 번호, 사주 기반 번호 추천 (메뉴 자리만 "준비중")
- 지도 표시 / 좌표 변환
- 서버리스 함수 기반 실시간 조회
